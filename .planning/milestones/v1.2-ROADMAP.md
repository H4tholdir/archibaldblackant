# Milestone v1.2: Production Parity

**Status:** SHIPPED 2026-02-24
**Phases:** 11-16
**Total Plans:** 14

## Overview

Chiusura di tutti i 25 gap trovati nell'audit 1:1 master vs branch — bootstrap production entry point, subclient system, fresis history completion, price/VAT Excel import, admin session & SSE, sync enhancements con checkpoint/resume, retry, delta sync e optimizer. Design doc: `docs/plans/2026-02-23-full-parity-v1.2-design.md`

## Phases

### Phase 11: Bootstrap & Production Entry Point
**Goal**: L'app deve poter partire in produzione — entry point, migrations, shutdown, background services.
**Depends on**: v1.1 complete
**Plans**: 2 plans

Plans:
- [x] 11-01: main.ts entry point (pool, queue, browserPool, scheduler, createApp, listen, shutdown)
- [x] 11-02: Migration runner on startup + background services init

**Details:**
- server.ts exports createApp(deps) — main.ts is bootstrap caller
- Graceful shutdown with signal handlers (SIGTERM, SIGINT)
- Migration runner executes on startup before app listen

### Phase 12: Subclient System
**Goal**: Subclient CRUD + import Excel completamente funzionanti con tutti i 15 campi dal master.
**Depends on**: Phase 11
**Plans**: 2 plans

Plans:
- [x] 12-01: Subclient PostgreSQL repository + wire stubs in server.ts
- [x] 12-02: Subclient Excel importer (code normalization, header mapping, bulk upsert)

**Details:**
- Single source of truth type in repository, imported by routes
- ILIKE search across ragione_sociale, suppl_ragione_sociale, codice
- Bulk upsert with ON CONFLICT (codice) DO UPDATE
- Excel header matching: case-insensitive lookup map with O(1) resolution
- normalizeSubClientCode: regex strips C prefix only when followed by digit
- Reconciliation: full sync semantics — DB records not in imported Excel are deleted

### Phase 13: Fresis History Completion
**Goal**: Tutte le funzionalità fresis — Arca export/import, FT numbering, endpoint mancanti, bulk discounts.
**Depends on**: Phase 11
**Plans**: 2 plans

Plans:
- [x] 13-01: getNextFtNumber + exportArca + importArca implementations
- [x] 13-02: Missing endpoints (reassign-merged, PUT /:id, archive) + discount bulk upload

**Details:**
- Atomic FT counter: INSERT ON CONFLICT DO UPDATE RETURNING for sequential numbering
- JSONB-text cast needed when passing DB rows to arca-export-service JSON.parse
- ft_counter seeded from imported NUMERODOC max via GREATEST()

### Phase 14: Price/VAT Excel Import
**Goal**: Import Excel VAT con matching prodotti e propagazione VAT ai sibling variants.
**Depends on**: Phase 11
**Plans**: 2 plans

Plans:
- [x] 14-01: Foundation — Migration + excel-vat-imports repository + product helpers (findSiblingVariants, updateProductVat)
- [x] 14-02: Excel VAT importer service + wire stubs in server.ts

**Details:**
- findSiblingVariants returns ALL siblings including self — caller filters
- escapeRegex local helper for dot-containing product IDs
- parseVatValue handles Italian comma format and percentage strings
- Invalid Excel buffers produce "missing required columns" error

### Phase 15: Admin Session & SSE
**Goal**: Impersonificazione admin funzionante + eventi SSE real-time per job progress.
**Depends on**: Phase 11
**Plans**: 2 plans

Plans:
- [x] 15-01: Admin sessions PostgreSQL repository + wire stubs in server.ts
- [x] 15-02: SSE job event bus + wire onJobEvent in server.ts/main.ts

**Details:**
- No getSession/getActiveSessions for admin sessions — YAGNI
- Factory function pub/sub for SSE event bus — no class needed
- Dual broadcast: main.ts fans out to both WebSocket and SSE event bus

### Phase 16: Sync Enhancements
**Goal**: Parità completa sistema sync — checkpoint/resume a livello pagina, retry con backoff, delta sync, optimizer.
**Depends on**: Phase 11
**Plans**: 3 plans

Plans:
- [x] 16-01: Sync checkpoint repository with page-level resume
- [x] 16-02: Product sync retry + delta sync endpoints
- [x] 16-03: Slowdown optimizer + smart sync variants

**Details:**
- Pure deriveResumePoint helper for threshold logic — 30min stale lock, 1h completion cooldown
- Checkpoint supplements sync_sessions, does not replace it
- sync_versions table separate from existing sync_metadata — avoids migration conflict
- Atomic version increment via UPDATE RETURNING inside transaction
- 1000-entry default limit with hasMore pagination for delta sync queries
- Binary search optimizer as separate factory function
- Sync modes via query parameter (?mode=) for backward compatibility

## Milestone Summary

**Audit Findings (25 gaps — ALL CLOSED):**
- 13 stubs in server.ts (subclients, prices, admin, fresis, SSE)
- 3 frontend endpoints without backend (fresis reassign-merged, PUT, archive)
- 4 infrastructure gaps (bootstrap, migrations, shutdown, background services)
- 3 sync system gaps (checkpoint/resume, retry, delta sync)
- 2 minor features (bulk discount upload, slowdown optimizer)

**Key Decisions:**
- server.ts exports createApp(deps) — main.ts bootstrap caller
- Operation processor wired with 10 handler types via handler map pattern
- Subclient type: single source of truth in repository
- Atomic FT counter with INSERT ON CONFLICT pattern
- Factory function pub/sub for SSE (no class)
- Binary search optimizer complementary to AdaptiveTimeoutManager

**Issues Resolved:**
- All 13 server.ts stubs replaced with real implementations
- Production bootstrap complete (main.ts, migrations, shutdown, background services)
- Full subclient CRUD + Excel import
- Complete fresis history system
- Admin session impersonation + SSE real-time events
- Sync checkpoint/resume, retry, delta sync, optimizer

**Issues Deferred:**
- None — full production parity achieved

**Technical Debt Incurred:**
- recordPriceChange .then(() => {}) pattern for Promise<void> deps type
- Some sync sessions logic duplicated between checkpoint and sync_sessions tables
