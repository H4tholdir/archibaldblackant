**Key architectural changes to verify don't affect behavior:**
1. Singleton → Dependency Injection (instances passed as args)
2. SQLite (better-sqlite3) → PostgreSQL (pg pool)
3. Monolithic index.ts → Modular route files
4. Direct execution → BullMQ operation queue
5. Custom sync scheduling → SyncScheduler service

**Audit approach:**
For each redesigned element from AUDIT-MATRIX.md:
1. Read the master implementation (`git show master:archibald-web-app/backend/src/index.ts` or route file)
2. Read the branch implementation (corresponding route file)
3. Compare: request validation, business logic, response shape, error handling
4. Flag divergences with severity
