- `updateOrderState(pool, userId, orderId, newState, actor, notes, confidence, source)` in orders repository
- updateOrderState does TWO things: (1) UPDATE order_records.current_state + last_sync, (2) INSERT into order_state_history
- send-to-verona.ts uses pool.query directly for raw SQL
- Operation handlers receive (pool, bot, data, userId, onProgress)

**Constraining decisions:**
- order_state_history table exists with schema: id, order_id, user_id, old_state, new_state, actor, notes, confidence, source, timestamp, created_at
- updateOrderState fetches current state first (SELECT), then updates (UPDATE), then inserts history (INSERT) — 3 queries total
- send-to-verona also needs to set sent_to_milano_at which updateOrderState doesn't handle — requires separate query
