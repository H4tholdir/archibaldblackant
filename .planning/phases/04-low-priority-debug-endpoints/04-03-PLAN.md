  1. Need access to a bot instance. In the branch, BrowserPool manages bots.

<task type="auto">
  <name>Task 2: Implement 6 health check PDF parser endpoints</name>
  <files>archibald-web-app/backend/src/server.ts</files>
  <action>
Add 6 GET /api/health/pdf-parser-* endpoints in server.ts createApp function. These are monitoring endpoints for checking availability of each PDF parser.

All health checks are unauthenticated (monitoring probes) — place them near the existing /api/health endpoint in server.ts.

Import the 6 services at the top of server.ts:
- `import { pdfParserService } from './pdf-parser-service'`
- `import { PDFParserProductsService } from './pdf-parser-products-service'`
- `import { PDFParserPricesService } from './pdf-parser-prices-service'`
- `import { PDFParserOrdersService } from './pdf-parser-orders-service'`
- `import { PDFParserDDTService } from './pdf-parser-ddt-service'`
- `import { PDFParserInvoicesService } from './pdf-parser-invoices-service'`

Implementation (match master behavior exactly):

1. **GET /api/health/pdf-parser** (generic customers parser):
   - `const isHealthy = await pdfParserService.healthCheck()`
   - 200: `{ status: "ok", message: "PDF parser ready (Python3 + PyPDF2 available)" }`
   - 503: `{ status: "error", message: "PDF parser not ready. Check logs for details." }`

2. **GET /api/health/pdf-parser-products**:
   - `const health = await PDFParserProductsService.getInstance().healthCheck()`
   - 200 if `health.healthy`: return health object
   - 503 if not healthy: return health object

3. **GET /api/health/pdf-parser-prices**:
   - `const health = await PDFParserPricesService.getInstance().healthCheck()`
   - 200 if healthy: `{ status: "ok", message: "Prices PDF parser ready...", ...health }`
   - 503 if not: `{ status: "unavailable", message: "Prices PDF parser not ready...", ...health }`

4. **GET /api/health/pdf-parser-orders** (sync, no await):
   - `const parserService = PDFParserOrdersService.getInstance()`
   - `const health = { available: parserService.isAvailable(), parser: "parse-orders-pdf.py", timeout: "300s", maxBuffer: "20MB" }`
   - 200 if available: `{ success: true, ...health }`
   - 503 if not: `{ success: false, message: "Orders PDF parser not available", ...health }`

5. **GET /api/health/pdf-parser-ddt** (sync, no await):
   - Same pattern as orders but: parser: "parse-ddt-pdf.py", timeout: "180s", maxBuffer: "20MB"

6. **GET /api/health/pdf-parser-invoices** (sync, no await):
   - Same pattern as orders but: parser: "parse-invoices-pdf.py", timeout: "120s", maxBuffer: "20MB"

Error handling: wrap async endpoints (1-3) in try/catch, return 500. Sync endpoints (4-6) are simple enough to not need try/catch.
