  <action>
Add 3 adaptive timeout endpoints directly in server.ts createApp function (they don't belong to a specific domain router — they're standalone like /metrics).

Implementation for all 3 endpoints:

1. **GET /api/timeouts/stats** (no auth in master — debug endpoint):
   - Import `AdaptiveTimeoutManager` from `./adaptive-timeout-manager`
   - `const manager = AdaptiveTimeoutManager.getInstance()`
   - `const stats = manager.getAllStats()`
   - Response: `{ success: true, data: stats }`
   - Error: 500 with error message

2. **POST /api/timeouts/reset/:operation?** (no auth in master):
   - Optional `operation` param from `req.params.operation`
   - If operation: `manager.resetStats(operation)` → message "Statistiche per {operation} resettate"
   - If no operation: `manager.resetStats()` → message "Tutte le statistiche timeout resettate"
   - Response: `{ success: true, message: "..." }`
   - Log the reset action with logger

3. **POST /api/timeouts/set** (no auth in master):
   - Body: `{ operation: string, timeout: number }`
   - Validate both present, timeout is number → 400 if missing
   - `manager.setTimeout(operation, timeout)`
   - Response: `{ success: true, message: "Timeout per {operation} impostato a {timeout}ms" }`
   - Error: 400 for validation, 500 for other errors

Place these 3 routes after /metrics and before the authenticated API routes.
Note: Master has NO authentication on these endpoints (debug/admin tools). Match this behavior.
