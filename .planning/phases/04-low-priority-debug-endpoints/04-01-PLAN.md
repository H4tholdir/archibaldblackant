  <action>
Add GET /api/cache/export endpoint in server.ts createApp function. This endpoint exports all data for frontend offline cache population.

Implementation:
- Add `app.get('/api/cache/export', authenticateJWT, ...)` in server.ts
- Extract userId from `(req as AuthRequest).user!.userId`
- Fetch data in parallel using Promise.all:
  1. `customersRepo.getCustomers(pool, userId)` — user-scoped in PostgreSQL branch (unlike master's global getAllCustomers)
  2. `productsRepo.getAllProducts(pool)` — global
  3. `productsRepo.getAllProductVariants(pool)` — global
  4. `pricesRepo.getAllPrices(pool)` — global
- Response shape (match master exactly):
  ```json
  {
    "success": true,
    "data": { "customers": [...], "products": [...], "variants": [...], "prices": [...] },
    "metadata": {
      "exportedAt": "ISO timestamp",
      "recordCounts": { "customers": N, "products": N, "variants": N, "prices": N }
    }
  }
  ```
- Log duration and record counts with logger
- Error: 500 with `{ success: false, error: "Cache export failed" }`
- Repos already imported in server.ts (customersRepo, productsRepo, pricesRepo) — just use them

Key difference from master: customers are user-scoped (getCustomers with userId) instead of global getAllCustomers. This is correct for the PostgreSQL branch where each agent has their own customer data.
  </action>
