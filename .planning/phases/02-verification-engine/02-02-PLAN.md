Creare una funzione `performInlineOrderSync` che esegue il sync articoli inline (non come job BullMQ).

**Signature:**
```typescript
type InlineSyncDeps = {
  pool: DbPool;
  downloadOrderArticlesPDF: (archibaldOrderId: string) => Promise<string>;
  parsePdf: (pdfPath: string) => Promise<ParsedArticle[]>;
  getProductVat: (articleCode: string) => Promise<number>;
  cleanupFile: (filePath: string) => Promise<void>;
};

async function performInlineOrderSync(
  deps: InlineSyncDeps,
  orderId: string,
  userId: string,
  onProgress: (progress: number, label?: string) => void,
): Promise<ParsedArticle[] | null>
```

**Logica:**
1. `onProgress(72, 'Sincronizzazione articoli...')`
2. Download PDF con retry (3 tentativi, delay 5s/10s/20s):
   - Se tutti i tentativi falliscono, return null (fallback a scheduler)
   - Usare `downloadOrderArticlesPDF(orderId)` — l'orderId restituito da createOrder è l'archibaldOrderId
3. `onProgress(78, 'Analisi articoli...')`
4. Parsare PDF via `parsePdf(pdfPath)`
5. Enrichire con VAT (stessa logica di sync-order-articles)
6. Salvare articoli in DB (DELETE + INSERT, stessa logica di sync-order-articles)
7. Aggiornare `articles_synced_at` in order_records (previene doppio sync da scheduler)
8. `onProgress(84, 'Articoli sincronizzati')`
9. Cleanup PDF file
10. Return parsed articles per il confronto successivo

**La logica di save articoli (step 6-7) va estratta o copiata da sync-order-articles.ts** — riusare lo stesso SQL per coerenza. NON chiamare handleSyncOrderArticles direttamente (ha dipendenze diverse).

**Se il download fallisce dopo tutti i retry:**
- Loggare warning
- Return null — il submit continua normalmente
- Lo scheduler farà il sync + verifica in seguito

**A. Estendere `handleSubmitOrder` (submit-order.ts):**

Dopo la transazione DB e il `onProgress(100, 'Ordine completato')` esistente, aggiungere il flusso di verifica per ordini non-warehouse:

```typescript
// DOPO la transazione DB (riga ~217)
// La progress bar va ridistribuita:
// - Tutto l'attuale 0-100% diventa 0-70% (scalare i valori nel BOT_PROGRESS_MAP e nei progress inline)
// - 70-85%: sync inline
// - 85-95%: verifica
// - 95-100%: risultato

if (!isWarehouseOnly && inlineSyncDeps) {
  // 1. Sync inline
  onProgress(70, 'Sincronizzazione articoli da Archibald...');
  const syncedArticles = await performInlineOrderSync(
    inlineSyncDeps, orderId, userId, onProgress
  );

  if (syncedArticles) {
    // 2. Verifica
    onProgress(87, 'Verifica ordine in corso...');
    const snapshot = await getOrderVerificationSnapshot(pool, orderId, userId);
    if (snapshot) {
      const result = verifyOrderArticles(snapshot.items, syncedArticles);

      // 3. Aggiorna status
      await updateVerificationStatus(
        pool, orderId, userId,
        result.status,
        result.mismatches.length > 0 ? JSON.stringify(result.mismatches) : null,
      );

      onProgress(95, result.status === 'verified'
        ? 'Ordine verificato correttamente'
        : 'Discrepanze rilevate nell\'ordine');
    }
  } else {
    onProgress(95, 'Verifica posticipata');
  }
}

onProgress(100, 'Ordine completato');
```
