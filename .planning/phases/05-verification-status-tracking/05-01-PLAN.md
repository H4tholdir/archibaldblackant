**Established patterns:**
- Order records in `agents.order_records`, articles in `agents.order_articles`
- Verification data in `agents.order_verification_snapshots` (status + notes JSON)
- Order types have ~41 fields; extend carefully
- OrderCardNew.tsx is 140KB+ — add badge minimally
- Inline style={{}} for all components
- GET /api/orders returns Order[] without articles; articles fetched separately
- sync-order-states updates `current_state` but does NOT touch verification data
- sync-order-articles replaces articles but does NOT touch verification snapshots

**Constraining decisions:**
- Verification snapshots are immutable (created once at submit)
- Badge persists until order state changes to inviato_milano/spedito/fatturato or is cancelled
- No manual "resolve" button — auto-resolution only via state change
- No dashboard — only per-order badge
- Sync never overwrites verification status
</context>

       - Only updates if current status is `correction_failed` or `mismatch_detected`

    2. **Call clearVerificationFlag in sync-order-states** when order state changes:
       - After `detectOrderState()` returns a new state, if the new state indicates the order has progressed past "piazzato":

       - Do NOT clear on state `piazzato`, `creato`, `modifica` — these are pre-send states where the user can still fix the order

       - When an order is hidden (via hidden_orders), the badge naturally disappears from the list
