```sql
CREATE TABLE IF NOT EXISTS agents.user_devices (
  id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL REFERENCES agents.users(id),
  device_identifier TEXT NOT NULL,
  platform TEXT NOT NULL,
  device_name TEXT NOT NULL,
  last_seen BIGINT NOT NULL,
  created_at BIGINT NOT NULL,
  UNIQUE(user_id, device_identifier)
);
```

Create `devices.ts` following the exact pattern of `users.ts`:

**Types:**
- `UserDevice` (camelCase): id, userId, deviceIdentifier, platform, deviceName, lastSeen, createdAt
- `UserDeviceRow` (snake_case): id, user_id, device_identifier, platform, device_name, last_seen, created_at
- `mapRowToDevice(row: UserDeviceRow): UserDevice`

**Functions (all accept DbPool as first arg):**

1. `registerDevice(pool, userId, deviceIdentifier, platform, deviceName): Promise<UserDevice>`
   - INSERT ON CONFLICT (user_id, device_identifier) DO UPDATE SET last_seen, platform, device_name
   - Use RETURNING to get the full row back
   - Generate id with crypto.randomUUID()
   - Set last_seen = Date.now(), created_at = Date.now()
   - Idempotent: same user+device updates metadata, new user+device creates row

2. `getUserDevices(pool, userId): Promise<UserDevice[]>`
   - SELECT * FROM agents.user_devices WHERE user_id = $1 ORDER BY last_seen DESC

3. `deleteDevice(pool, deviceId): Promise<void>`
   - DELETE FROM agents.user_devices WHERE id = $1

4. `cleanupOldDevices(pool, daysThreshold = 90): Promise<number>`
   - DELETE FROM agents.user_devices WHERE last_seen < $1 (Date.now() - days*86400000)
   - RETURNING id to get count of deleted rows
   - Return result.rowCount

**Tests in devices.spec.ts:**
Follow the mock-pool pattern from users.spec.ts. Test:
- registerDevice calls INSERT ON CONFLICT with correct params, maps result row
- getUserDevices returns mapped array sorted by last_seen DESC
- deleteDevice calls DELETE with device id
- cleanupOldDevices calculates threshold correctly, returns rowCount

Export all functions and types.

   Optional because DI pattern uses graceful degradation.

2. In the login handler, AFTER successful JWT generation (line ~83), add device registration:
   ```
   const { platform, deviceName } = parsed.data;
   if (deps.registerDevice && deviceId) {
     deps.registerDevice(user.id, deviceId, platform || 'unknown', deviceName || 'Unknown Device')
       .catch((err) => logger.warn('Failed to register device', { userId: user.id, deviceId, error: err }));
   }
   ```
   NON-FATAL: fire-and-forget with catch for warning log. Device registration failure must never block login.
