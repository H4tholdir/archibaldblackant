**Established patterns:**
- Pure functions for verification logic (no DB deps)
- VerificationStatus: 'pending_verification' | 'verified' | 'mismatch_detected' | 'auto_corrected' | 'correction_failed'
- ArticleMismatch: { type, snapshotArticleCode, syncedArticleCode, field, expected, found }
- MismatchType: 'missing' | 'extra' | 'quantity_diff' | 'price_diff' | 'discount_diff' | 'amount_diff'

**Constraining decisions:**
- Phase 3: price_diff is uncorrectable (bot can't change unit price)
- Phase 3: One correction attempt only, then correction_failed
- Phase 4 vision: silenzioso se OK, notifica solo per correction_failed o mismatch_detected non corretto
</context>

<feature>
  <name>formatVerificationNotification</name>
  <files>archibald-web-app/backend/src/verification/format-notification.ts, archibald-web-app/backend/src/verification/format-notification.spec.ts</files>
  <behavior>
    Pure function: (status: VerificationStatus, mismatches: ArticleMismatch[]) => VerificationNotification | null

    Returns null for statuses that don't need notification (verified, auto_corrected, pending_verification).

    For correction_failed or mismatch_detected, returns:
    {
      status: VerificationStatus,
      summary: string,           // e.g. "3 discrepanze rilevate nell'ordine"
      items: NotificationItem[], // One per mismatch, with human-readable detail
      severity: 'warning' | 'error'
    }

    NotificationItem = {
      articleCode: string,
      message: string,           // e.g. "Quantità diversa: atteso 5, trovato 3"
      type: MismatchType,
      expected: number | null,
      found: number | null
    }

    Cases:
    - 'verified' → null
    - 'auto_corrected' → null
    - 'pending_verification' → null
    - 'mismatch_detected' with 1 missing article → { summary: "1 discrepanza rilevata...", items: [{ message: "Articolo mancante nell'ordine Archibald" }], severity: 'warning' }
    - 'correction_failed' with mixed mismatches → { summary: "N discrepanze...", items: [...], severity: 'error' }
    - 'mismatch_detected' with quantity_diff → item.message: "Quantità diversa: atteso X, trovato Y"
    - 'mismatch_detected' with price_diff → item.message: "Prezzo unitario diverso: atteso X €, trovato Y €"
    - 'mismatch_detected' with discount_diff → item.message: "Sconto diverso: atteso X%, trovato Y%"
    - 'mismatch_detected' with amount_diff → item.message: "Importo diverso: atteso X €, trovato Y €"
    - 'mismatch_detected' with extra article → item.message: "Articolo extra non previsto nell'ordine"
    - Multiple mismatches on same article → separate items, one per mismatch
    - Empty mismatches with mismatch_detected status → still returns notification with summary "Discrepanze rilevate" and empty items
  </behavior>
  <implementation>
    Import VerificationStatus from order-verification.ts and ArticleMismatch/MismatchType from verify-order-articles.ts.
    Export branded types VerificationNotification and NotificationItem.
    Use a switch/map for MismatchType → Italian message template.
    Format currency values with 2 decimals and € symbol.
    Format percentage values with % suffix.
    Summary uses singular/plural: "1 discrepanza rilevata" vs "N discrepanze rilevate".
    severity: 'error' for correction_failed, 'warning' for mismatch_detected.
  </implementation>
</feature>
