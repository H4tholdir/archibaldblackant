**Established patterns:**
- WebSocket broadcast: broadcast(userId, { event, jobId, type, ...data })
- Transient events (JOB_PROGRESS) not buffered; non-transient events buffered for replay
- usePendingSync handles JOB_COMPLETED by refetching after 4s
- Express routes use factory functions: createXxxRouter(deps)
- Inline style={{}} for components (no CSS modules)
- fetchWithRetry for API calls

**Constraining decisions:**
- Silenzioso se OK (verified/auto_corrected → no notification)
- Notifica solo per correction_failed o mismatch_detected non corretto
- Dual delivery: WebSocket real-time + DB query on page load
- Notification vive sulla card del pending order, non in centro notifiche
- Progress bar already handles 70-100% verification range
- verification_notes in DB already stores JSON mismatches — no new migration needed
