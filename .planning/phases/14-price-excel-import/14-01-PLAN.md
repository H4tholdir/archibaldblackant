**Master excel-vat-importer.ts Details:**
- importFromExcel(pool, buffer, filename, userId, productsRepo):
  1. Parse Excel with xlsx
  2. For each row: match product by ID first, fallback to normalized Codice Articolo
  3. Update VAT if changed
  4. Optionally update price if column present
  5. Propagate VAT to sibling variants (same base code, different K/R suffix)
  6. Record each change via prices-history repo (audit trail)
  7. Record import summary in excel_vat_imports table
  8. Return { totalRows, matched, unmatched, vatUpdated, priceUpdated }

**Sibling Variant Logic:**
- Products with codes like "ABC123K" and "ABC123R" are siblings (same base "ABC123")
- When VAT changes on one variant, propagate to all siblings
- Suffix patterns: K (kit), R (ricambio), or no suffix (base)

    Import flow:
    1. Parse Excel buffer using xlsx library
    2. Identify columns: ID, Codice Articolo, IVA (%), Prezzo (optional)

       b. If no match, try normalized Codice Articolo -> deps.findProductByArticolo
       c. If no match, count as unmatched, continue

       f. If price column present and price changed: update price
       g. Record each change via deps.recordPriceChange (audit)

    5. Return ImportVatResult
