    **Function 1: findSiblingVariants(pool, productId)**

    Purpose: Given a product ID like "ABC123K", find all sibling variants that share the same base code (strip trailing K or R suffix). Siblings include: "ABC123", "ABC123K", "ABC123R".

    Implementation:
    ```typescript
    function extractBaseCode(productId: string): string {
      return productId.replace(/[KR]$/i, '');
    }

    async function findSiblingVariants(
      pool: DbPool,
      productId: string,
    ): Promise<ProductRow[]> {
      const baseCode = extractBaseCode(productId);
      const { rows } = await pool.query<ProductRow>(
        `SELECT * FROM shared.products
         WHERE id ~ $1 AND deleted_at IS NULL`,
        [`^${escapeRegex(baseCode)}[KRkr]?$`],
      );
      return rows;
    }
    ```

    IMPORTANT: Escape regex special chars in baseCode (product IDs contain dots like "h129fsq.104.023"). Create a simple `escapeRegex` helper that escapes `.`, `(`, `)`, etc. This helper does NOT need its own file — define it locally in products.ts since it's only used here.

    Exclude the queried product itself from results? No — return ALL siblings including self. The caller (excel-vat-importer service) will filter out the source product.

    **Function 2: updateProductVat(pool, productId, vat, vatSource)**

    Purpose: Update ONLY the VAT fields on a product, without touching price fields. The existing `updateProductPrice` requires a price value, which we don't have during VAT-only imports.

    ```typescript
    async function updateProductVat(
      pool: DbPool,
      productId: string,
      vat: number,
      vatSource: string,
    ): Promise<boolean> {
      const { rowCount } = await pool.query(
        `UPDATE shared.products SET
           vat = $2,
           vat_source = $3,
           vat_updated_at = NOW(),
           updated_at = NOW()
         WHERE id = $1`,
        [productId, vat, vatSource],
      );
      return (rowCount ?? 0) > 0;
    }
    ```

    Export both functions and `extractBaseCode` (pure function, useful for testing).

    **Tests (write FIRST):**

    `describe('findSiblingVariants')`:
    - Returns all products sharing base code (e.g., "ABC123" finds "ABC123", "ABC123K", "ABC123R")
    - Returns empty array when no siblings exist
    - Handles product IDs with dots correctly (e.g., "h129.001" doesn't match "h129.0010")
    - Excludes soft-deleted products (deleted_at IS NOT NULL)

    `describe('extractBaseCode')`:
    - Strips trailing K: "ABC123K" → "ABC123"
    - Strips trailing R: "ABC123R" → "ABC123"
    - No change for base codes: "ABC123" → "ABC123"
    - Handles lowercase: "abc123k" → "abc123"
    - Does NOT strip K/R from middle: "KR123" → "KR123" (only strips trailing)

    `describe('updateProductVat')`:
    - Updates VAT and vat_source on existing product
    - Returns true when product exists
    - Returns false when product doesn't exist
    - Does not touch price fields
