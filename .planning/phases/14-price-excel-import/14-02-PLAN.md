**Key contracts:**
- Route expects `importExcel: (buffer: Buffer, filename: string, userId: string) => Promise<ImportResult>`
- ImportResult = `{ totalRows: number, matched: number, unmatched: number, errors: string[] }`
- Route expects `getImportHistory: () => Promise<ImportRecord[]>`
- ImportRecord = `{ id, filename, uploadedBy, uploadedAt, totalRows, matched, unmatched, status }`

**Sibling variant logic (from master):**
- Products with codes like "ABC123K" and "ABC123R" are siblings (same base "ABC123")
- When VAT changes on one variant, propagate to all siblings
- Suffix patterns: K (kit), R (ricambio), or no suffix (base)
- extractBaseCode (from 14-01) strips trailing K/R
- findSiblingVariants (from 14-01) returns all products sharing base code

    **Excel parsing (follow subclient-excel-importer.ts pattern):**

    Expected Excel columns with case-insensitive header variations:
    ```typescript
    const HEADER_VARIATIONS = {
      productId: ['ID', 'Id', 'id', 'Product ID', 'Codice', 'CODICE', 'Codice Articolo', 'CODICE ARTICOLO'],
      vat: ['IVA', 'Iva', 'iva', 'IVA %', 'IVA%', 'VAT', 'Aliquota IVA'],
      price: ['Prezzo', 'PREZZO', 'Price', 'Prezzo Unitario', 'PREZZO UNITARIO'],

    ```

    Build case-insensitive header lookup map (same pattern as subclient-excel-importer).

    **Import algorithm:**

    ```
    importExcelVat(buffer, filename, userId, deps):
      1. Parse Excel: xlsx.read(buffer, { type: 'buffer' })
      2. Get first sheet, sheet_to_json with header:1
      3. Map headers to canonical fields (productId, vat, price)
      4. If no productId column found → return error
      5. If no vat column found → return error
      6. For each data row:
         a. Extract productId, vat, price (optional) from row using column map
         b. Skip row if productId is empty/null → count as unmatched
         c. Parse VAT as number (handle Italian comma format: "22,00" → 22)
         d. Lookup product: deps.getProductById(productId)
         e. If not found → add to errors, count as unmatched, continue
         f. If VAT changed (product.vat !== newVat):
            - deps.updateProductVat(productId, newVat, 'excel-import')
            - Record audit: deps.recordPriceChange with changeType and source='excel-vat-import'
            - Propagate: deps.findSiblingVariants(productId)
              For each sibling where sibling.id !== productId:
                deps.updateProductVat(sibling.id, newVat, 'excel-import-propagated')
                deps.recordPriceChange for sibling too
            - Increment vatUpdated counter
         g. If price column present AND price value exists AND price changed:
            - deps.updateProductPrice(productId, newPrice, newVat, 'excel-import', 'excel-import')
            - Record audit via deps.recordPriceChange
            - Increment priceUpdated counter
         h. Increment matched counter
      7. Record import summary: deps.recordImport({ filename, uploadedBy: userId, totalRows, matched, unmatched, vatUpdated, priceUpdated, status })
      8. Return { totalRows, matched, unmatched, errors }
    ```

    **Helper: parseVatValue(value: unknown): number | null**
    - Handles numeric (22), string ("22"), Italian format ("22,00"), percentage string ("22%")
    - Returns null for unparseable values

    **Tests (write FIRST):**

    `describe('parseVatValue')`:
    - Parses integer: 22 → 22
    - Parses string: "22" → 22
    - Parses Italian format: "22,00" → 22
    - Parses with percent: "22%" → 22
    - Returns null for empty/null/undefined
    - Returns null for non-numeric strings

    `describe('importExcelVat')`:
    Use mock deps (vi.fn()) — this is a pure-logic unit test with no DB.

    - **Happy path**: Excel with 3 rows, all products found, VAT changed on 2 → returns { totalRows: 3, matched: 3, unmatched: 0, errors: [] }, vatUpdated: 2
    - **Product not found**: Row with unknown product ID → unmatched + 1, error message added
    - **VAT unchanged**: Product already has same VAT → matched but no update call
    - **Sibling propagation**: Product "ABC123K" VAT changes → findSiblingVariants called → updateProductVat called for "ABC123" and "ABC123R" too
    - **Price column present**: When price column exists and value changed → updateProductPrice called
    - **Price column absent**: When no price column → only VAT updates, no price updates
    - **Empty Excel**: No data rows → { totalRows: 0, matched: 0, unmatched: 0, errors: [] }
    - **Invalid buffer**: Non-Excel buffer → errors contains parse error message
    - **Missing required columns**: No IVA column → errors contains column error
    - **Audit trail**: For each VAT change, recordPriceChange called with source='excel-vat-import'
    - **Audit trail for siblings**: For each propagated change, recordPriceChange called with source='excel-import-propagated'
    - **recordImport called**: After processing, recordImport called with correct stats

    Create test helpers:
    - `createMockDeps()`: Returns deps object with vi.fn() for each function
    - `createTestExcelBuffer(rows)`: Creates real xlsx buffer from row data for realistic testing
