  <action>
    Create migration file following the pattern of 002-shared-tables.sql.

    Table: shared.price_history with columns:
    - id SERIAL PRIMARY KEY
    - product_id TEXT NOT NULL
    - product_name TEXT NOT NULL
    - variant_id TEXT (itemSelection: K2, K3, etc.)
    - old_price TEXT (Italian format, as stored in shared.prices.unit_price — keep as TEXT for consistency)
    - new_price TEXT NOT NULL (same format)
    - old_price_numeric DOUBLE PRECISION (parsed number for calculations)
    - new_price_numeric DOUBLE PRECISION NOT NULL
    - price_change DOUBLE PRECISION (absolute: new - old)
    - percentage_change DOUBLE PRECISION (percentage: (new-old)/old*100)
    - change_type TEXT NOT NULL CHECK (change_type IN ('increase', 'decrease', 'new'))
    - source TEXT NOT NULL (e.g. 'pdf-sync', 'excel-upload', 'manual')
    - currency TEXT
    - changed_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()

    Indexes:
    - idx_price_history_product ON (product_id)
    - idx_price_history_changed_at ON (changed_at)
    - idx_price_history_product_variant ON (product_id, variant_id)
    - idx_price_history_recent ON (changed_at DESC, percentage_change DESC) — for dashboard queries

    NOTE: Use TEXT for old_price/new_price (matching shared.prices.unit_price format) + DOUBLE PRECISION for numeric versions (for aggregation queries). This avoids format conversion at query time.
  </action>
  <verify>Check migration SQL is syntactically valid by reviewing structure</verify>

  <action>
    Create repository following the pattern of prices.ts repository (pure functions taking DbPool).

    Export these functions:
    1. recordPriceChange(pool, data) — INSERT into shared.price_history. Data includes: productId, productName, variantId, oldPrice (text), newPrice (text), oldPriceNumeric, newPriceNumeric, priceChange, percentageChange, changeType, source, currency. Calculate changeType from old/new numeric prices.

    2. getProductHistory(pool, productId, limit?) — SELECT * WHERE product_id=$1 ORDER BY changed_at DESC LIMIT $2. Return type must match PriceHistoryEntry from routes/prices.ts (map DB snake_case to camelCase).

    3. getRecentChanges(pool, days) — SELECT WHERE changed_at >= NOW() - interval '$1 days' ORDER BY changed_at DESC, ABS(percentage_change) DESC.

    4. getRecentStats(pool, days) — Aggregate query returning PriceHistoryStats: COUNT(*) as totalChanges, SUM(CASE increase) as increases, SUM(CASE decrease) as decreases, SUM(CASE new) as newPrices, AVG(CASE increase THEN percentage_change) as avgIncrease, AVG(CASE decrease THEN percentage_change) as avgDecrease.

    5. getTopIncreases(pool, days, limit) — SELECT WHERE change_type='increase' AND changed_at >= cutoff ORDER BY percentage_change DESC LIMIT $3.

    6. getTopDecreases(pool, days, limit) — SELECT WHERE change_type='decrease' AND changed_at >= cutoff ORDER BY percentage_change ASC LIMIT $3.

    Export types:
    - PriceHistoryRow (DB row shape, snake_case)
    - PriceHistoryInsert (input for recordPriceChange)

    For unit tests: mock the pool.query calls (same pattern as prices.spec.ts). Test all 6 functions with expected SQL parameterization. Use describe('functionName', ...) grouping.
