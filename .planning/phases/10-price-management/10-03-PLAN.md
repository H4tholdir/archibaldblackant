  <files>archibald-web-app/backend/src/services/price-matching.ts, archibald-web-app/backend/src/services/price-matching.spec.ts</files>
  <action>
    Add matchPricesToProducts to the existing price-matching.ts file (which already has parseItalianPrice + matchVariant from Plan 10-01).

    Function signature:
    ```
    async function matchPricesToProducts(deps: MatchPricesToProductsDeps): Promise<MatchPricesToProductsResult>
    ```

    MatchPricesToProductsDeps type:
    - getAllPrices: () => Promise<PriceRow[]>
    - getProductVariants: (name: string) => Promise<ProductRow[]>
    - updateProductPrice: (id: string, price: number, vat: number | null, priceSource: string, vatSource: string | null) => Promise<boolean>
    - recordPriceChange: (data: PriceHistoryInsert) => Promise<void>

    Algorithm (port from master's PriceMatchingService):
    1. Get all prices via getAllPrices()
    2. For each price record:
       a. Skip if unit_price is null → count as nullPrices, add to unmatchedPrices with reason "null_price"
       b. Parse unit_price via parseItalianPrice() → skip if null (unparseable)
       c. Find matching products via getProductVariants(price.product_name)
       d. If no products found → count as unmatched, reason "product_not_found"
       e. Match variant via matchVariant(products, price.item_selection)
       f. If no variant match → count as variantMismatch, reason "variant_mismatch"
       g. Update product price via updateProductPrice(product.id, parsedPrice, product.vat, "prices-db", null)
       h. If product's old price differs from new → recordPriceChange() with source "pdf-sync"
    3. Return { result: MatchResult, unmatchedPrices: UnmatchedPrice[] }

    MatchResult type: { matched, unmatched, skipped } (matching PricesRouterDeps contract)
    UnmatchedPrice type: { productId, productName } (matching PricesRouterDeps contract)

    Unit tests: mock all deps, test the algorithm logic:
    - Price with null unit_price → skipped, added to unmatched
    - Price with no matching product → unmatched
    - Price with variant mismatch → unmatched
    - Price with match → product updated, price change recorded
    - Price unchanged (same price) → no history record
  </action>
  <verify>npm test --prefix archibald-web-app/backend -- --run price-matching</verify>
  <done>matchPricesToProducts function works correctly with all edge cases tested</done>
