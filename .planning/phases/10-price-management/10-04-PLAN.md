  <files>archibald-web-app/backend/src/operations/handlers/sync-prices.ts, archibald-web-app/backend/src/operations/handlers/sync-prices.spec.ts</files>
  <action>
    Create handler following the pattern of send-to-verona.ts.

    Define bot interface type:
    ```
    type SyncPricesBot = {
      downloadPricePdf: () => Promise<string>;  // returns path to downloaded PDF
    };
    ```

    Factory function:
    ```
    function createSyncPricesHandler(
      pool: DbPool,
      parsePdf: (pdfPath: string) => Promise<ParsedPrice[]>,
      cleanupFile: (filePath: string) => Promise<void>,
      createBot: (userId: string) => SyncPricesBot,
    ): OperationHandler
    ```

    Handler implementation:
    1. Create bot via createBot(userId)
    2. Call syncPrices({ pool, downloadPdf: () => bot.downloadPricePdf(), parsePdf, cleanupFile }, onProgress, () => false)
    3. Return the PriceSyncResult as Record<string, unknown>
    4. The shouldStop function returns false for now (no preemption support yet — can be added later)

    For tests: mock all deps, verify syncPrices is called with correct arguments, verify result is returned.

    NOTE: The bot.downloadPricePdf() method doesn't exist yet on ArchibaldBot — that's fine. The handler accepts a createBot factory via DI, so the actual bot implementation can be wired later. For now, the handler just needs to be structurally correct and testable. The server.ts wiring in Task 2 will provide a stub bot implementation that logs a warning.
