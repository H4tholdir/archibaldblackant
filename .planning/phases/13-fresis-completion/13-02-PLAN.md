**Frontend API contracts (MUST match exactly):**

1. **PUT /:id** — `updateFresisHistoryOrder(id, data: Partial<FresisHistoryOrder>)`
   - Request: PUT /api/fresis-history/:id with JSON body = Partial<FresisHistoryOrder> (camelCase)
   - Response: 200 OK (frontend doesn't parse response body)
   - Error: throw on non-200

2. **POST /reassign-merged** — `reassignMergedOrderId(oldMergedId, newMergedId)`
   - Request: POST /api/fresis-history/reassign-merged with `{ oldMergedId: string, newMergedId: string }`
   - Response: `{ count: number }` — frontend reads `data.count`
   - NOT `{ recordIds, newMotherOrderId }` as originally planned

3. **POST /archive** — `archiveOrders(orders, mergedOrderId?)`
   - Request: POST /api/fresis-history/archive with:
     ```typescript
     {
       orders: Array<{
         id: string;
         customerId: string;
         customerName: string;
         subClientCodice?: string;
         subClientName?: string;
         subClientData?: unknown;
         items: unknown[];
         discountPercent?: number;
         targetTotalWithVAT?: number;
         shippingCost?: number;
         shippingTax?: number;
         revenue?: number;
         createdAt: string;
       }>,
       mergedOrderId?: string
     }
     ```
   - Response: `{ records: FresisHistoryOrder[] }` — frontend reads `data.records`
   - This creates/upserts fresis history records from pending orders being merged
   - NOT a "mark as archived" operation — there is NO archived column in the schema
