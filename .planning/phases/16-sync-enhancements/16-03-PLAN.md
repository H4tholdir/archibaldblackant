    Create services/slowdown-optimizer.ts:

    1. **Function signature:**
       ```typescript
       type OptimizeResult = {
         optimalDelay: number;
         iterations: number;
         converged: boolean;
         crashCount: number;
       };

       type BotAction = (delayMs: number) => Promise<boolean>; // true = success, false = crash/error

       function createSlowdownOptimizer(options?: {
         minDelay?: number;     // Default: 0
         maxDelay?: number;     // Default: 200
         convergenceThreshold?: number; // Default: 5 (ms)
         maxIterations?: number; // Default: 50
         maxCrashes?: number;   // Default: 10
       })
       ```
       Returns `{ optimizeStep }`.

    2. **optimizeStep(stepName, action: BotAction) -> OptimizeResult:**
       Binary search algorithm:
       ```
       low = minDelay, high = maxDelay
       crashCount = 0, iterations = 0
       lastGoodDelay = maxDelay

       while (high - low > convergenceThreshold && iterations < maxIterations && crashCount < maxCrashes):
         mid = Math.round((low + high) / 2)
         success = await action(mid)
         iterations++

         if success:
           lastGoodDelay = mid
           high = mid        // Can we go lower?
         else:
           crashCount++
           low = mid         // Need more delay

       return { optimalDelay: lastGoodDelay, iterations, converged: high - low <= convergenceThreshold, crashCount }
       ```

    3. **Persist results** (optional integration with adaptive-timeouts.json):
       - After optimization, update the relevant operation's currentTimeout
       - Can integrate with AdaptiveTimeoutManager or store independently

    4. **Create comprehensive spec:**
       - Test binary search converges for always-succeeding action (finds minimum delay ~0)
       - Test binary search converges for action that fails below 100ms (finds ~100ms)
       - Test maxIterations limit stops search
       - Test maxCrashes limit stops search
       - Test convergenceThreshold determines when to stop
       - Use mock BotAction that succeeds/fails based on delay threshold
       - Test with varying success thresholds to verify binary search correctness
