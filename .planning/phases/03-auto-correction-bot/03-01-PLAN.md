  <name>buildCorrections ‚Äî mappa mismatches in bot modifications</name>

  <behavior>
    Funzione pura `buildCorrections(mismatches, snapshotItems, syncedArticles)` ‚Üí `CorrectionPlan`

    **Tipi:**
    - `CorrectionPlan`: { modifications: Modification[], updatedItems: EditOrderArticle[], canCorrect: boolean, uncorrectableReasons: string[] }
    - `Modification`: union type matching edit-order bot format (update/add/delete)

    **Input:** mismatches (da verifyOrderArticles), snapshotItems (dati corretti), syncedArticles (dati attuali con posizione/indice)

    **Mapping rules:**
    1. `missing` ‚Üí `{ type: 'add', articleCode, quantity, discount }` dal snapshot
    2. `extra` ‚Üí `{ type: 'delete', rowIndex }` ‚Äî rowIndex dall'ordine nell'array sincronizzato
    3. `quantity_diff` ‚Üí `{ type: 'update', rowIndex, articleCode, quantity: expected }`
    4. `discount_diff` ‚Üí `{ type: 'update', rowIndex, articleCode, quantity: current, discount: expected }`
    5. `price_diff` ‚Üí uncorrectable (bot non pu√≤ cambiare prezzo, segna `canCorrect: false` con reason)
    6. `amount_diff` senza qty/discount diff ‚Üí uncorrectable (differenza inspiegabile)
    7. Multipli mismatch sullo stesso articolo ‚Üí merge in una singola modification

    **Casi di test (RED phase):**
    1. Nessun mismatch ‚Üí modifications vuoto, canCorrect: true
    2. `missing` ‚Üí aggiunge riga con dati dallo snapshot
    3. `extra` ‚Üí cancella riga all'indice corretto
    4. `quantity_diff` ‚Üí update con quantit√Ý attesa
    5. `discount_diff` ‚Üí update con sconto atteso
    6. `price_diff` ‚Üí canCorrect: false con reason
    7. `amount_diff` isolato ‚Üí canCorrect: false con reason
    8. Multipli mismatch (qty + discount) stesso articolo ‚Üí singola update con entrambi
    9. Mix di tipi diversi ‚Üí modifications ordinati: updates prima, adds dopo, deletes ultime (per preservare indici)
    10. `updatedItems` ricostruito correttamente dallo snapshot per DB update post-correzione
  </behavior>
  <implementation>
    1. Creare `src/verification/build-corrections.ts`
    2. Funzione pura ‚Äî input i 3 array, output il piano di correzione
    3. Per ogni mismatch, trovare il rowIndex corrispondente nell'array syncedArticles
    4. Merge mismatch sullo stesso articolo in una singola modification
    5. Ordinare: updates ‚Üí adds ‚Üí deletes (deletes in ordine decrescente di rowIndex per non spostare indici)
    6. Se `price_diff` o `amount_diff` isolato ‚Üí `canCorrect: false`
    7. `updatedItems` = snapshot items mappati in EditOrderArticle format
  </implementation>
</feature>
