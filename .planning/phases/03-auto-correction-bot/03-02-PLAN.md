  <action>
Creare una funzione `performAutoCorrection` che orchestra: build corrections → bot edit → re-sync → re-verify.

**Signature:**
```typescript
type AutoCorrectionDeps = {
  pool: DbPool;
  editOrderInArchibald: (archibaldOrderId: string, modifications: Modification[]) => Promise<{ success: boolean; message: string }>;
  inlineSyncDeps: InlineSyncDeps;  // Per ri-sync dopo correzione
};

type AutoCorrectionResult = {
  status: 'auto_corrected' | 'correction_failed';
  details: string;  // JSON con dettagli
};

async function performAutoCorrection(
  deps: AutoCorrectionDeps,
  orderId: string,
  userId: string,
  mismatches: ArticleMismatch[],
  snapshotItems: SnapshotArticle[],
  syncedArticles: SyncedArticle[],
  onProgress: (progress: number, label?: string) => void,
): Promise<AutoCorrectionResult>
```

**Logica:**
1. `onProgress(90, 'Analisi correzioni...')`
2. Chiama `buildCorrections(mismatches, snapshotItems, syncedArticles)`
3. Se `!correctionPlan.canCorrect` → return `correction_failed` con reasons
4. `onProgress(91, 'Correzione ordine in corso...')`
5. Chiama `editOrderInArchibald(orderId, correctionPlan.modifications)`
6. Se bot fallisce → return `correction_failed` con messaggio errore
7. `onProgress(94, 'Ri-sincronizzazione articoli...')`
8. Chiama `performInlineOrderSync(deps.inlineSyncDeps, orderId, userId, onProgress)` — riusa la funzione di Phase 2
9. Se re-sync fallisce → return `correction_failed`
10. `onProgress(97, 'Ri-verifica ordine...')`
11. Leggi snapshot con `getOrderVerificationSnapshot`
12. Chiama `verifyOrderArticles(snapshot.items, reSyncedArticles)`
13. Se ri-verifica OK → return `auto_corrected`
14. Se ri-verifica ancora mismatch → return `correction_failed` con nuovi mismatch

**Error handling:** try/catch su ogni step. Se qualsiasi step fallisce, return `correction_failed`. MAI throw — il submit non deve mai essere bloccato.

**A. Estendere submit-order.ts:**

Dove attualmente la verifica trova `mismatch_detected` (dopo `updateVerificationStatus`), aggiungere il flusso di correzione:

```typescript
if (result.status === 'mismatch_detected' && autoCorrectionDeps) {
  onProgress(90, 'Correzione ordine in corso...');
  const correctionResult = await performAutoCorrection(
    autoCorrectionDeps,
    orderId, userId,
    result.mismatches,
    snapshot.items,
    syncedArticles,
    onProgress,
  );

  // Aggiorna lo status della verifica con il risultato della correzione
  await updateVerificationStatus(
    pool, orderId, userId,
    correctionResult.status,
    correctionResult.details,
  );

  verificationStatus = correctionResult.status;

  if (correctionResult.status === 'auto_corrected') {
    onProgress(99, 'Ordine corretto e verificato');
  } else {
    onProgress(99, 'Correzione non riuscita - intervento necessario');
  }
}
```

**B. Estendere la signature:**
Aggiungere parametro opzionale `autoCorrectionDeps?: AutoCorrectionDeps`.

**C. Wiring in main.ts:**
Passare le dipendenze per l'auto-correzione a `createSubmitOrderHandler`:
- `editOrderInArchibald`: factory che acquisisce browser context, chiama `bot.editOrderInArchibald`, rilascia context
- `inlineSyncDeps`: riusare le stesse deps già passate per il sync inline (Phase 2)

**D. Ridistribuire progress bar per il caso mismatch:**
Quando c'è mismatch e correzione:
